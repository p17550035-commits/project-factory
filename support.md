# Support Guide

This document explains how to get help with Project Factory, how to report problems, and how to request new features.

---

# Getting Help

If you need assistance:

- Review the README.md  
- Check existing documentation  
- Look at open issues  
- Search for similar questions  

If you still need help, you may open a support request.

---

# Reporting Issues

When reporting a problem:

- Describe the issue clearly  
- Include steps to reproduce  
- Provide logs or screenshots if possible  
- Mention your environment (OS, device, Python version)  
- Explain what you expected vs what happened  

This helps maintainers resolve issues faster.

---

# Requesting Features

Feature requests should include:

- A clear description of the feature  
- Why it would be useful  
- Any examples or mockups  
- Whether it affects existing behavior  

All feature requests will be reviewed and prioritized.

---

# Security Issues

For security-related concerns:

- Do **not** open a public issue  
- Contact the project owner privately  
- Provide as much detail as possible  

See SECURITY.md for full guidelines.

---

# Notes

Support processes may evolve as the project grows.  
Updates will be reflected in this document.
